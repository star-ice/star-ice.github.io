Python Encoding Format

Posted on March 24, 2019 本文总阅读量次

常见的编码类型


  1. ASCII码，用1个字节存储英文和字符，包含256个字符

  2. Unicode码，2个字节

  3. UTF-8，可以使用1~4个字节表示一个符号，根据不同的符号而变化字节长度。对于单字节的符号，字节的第一位设为0，后面7位为这个符号的unicode码。因此对于英语字母，UTF-8编码和ASCII码是相同的。而对于n字节的符号（n>1），第一个字节的前n位都设为1，第n+1位设为0，后面字节的前两位一律设为10。剩下的没有提及的二进制位，全部为这个符号的unicode码。

有时会遇到在.py文件中出现中文时，报SyntaxError: Non-ASCII character ‘\xe4’ in file t.py on line 4, but no encoding declared; see http://www.python.org/peps/pep-0263.html for details错误，这是因为在编译过程中无法识别非ASCII码造成的；当我们从外部读取中文内容时也常常会出现乱码，也经常会出现中文乱码的现象，这是因为中文字符串的编码和Python的编码不一致而造成的。为了解决这些中文编码问题，通常需要借助于Unicode来实现中文字符解码到编码指定输出编码类型的过程。
